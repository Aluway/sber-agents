# Идея: Персональный финансовый советник

## Концепция

Персональный финансовый советник в виде LLM-based Telegram-бота для автоматического учета доходов и расходов на основе данных из чата.

---

## Цель

Создать интеллектуального бота, который будет вести учет финансов пользователя, автоматически извлекая информацию о транзакциях из различных типов сообщений (текст, голос, изображения).

---

## Функционал

### 1. Таблица учета доходов и расходов

Бот ведет в памяти структурированную таблицу со следующими полями:

- **Дата** — дата транзакции
- **Время** — время транзакции
- **Доход/Расход** — тип операции
- **Сумма** — денежная сумма
- **Тип** — классификация по частоте:
  - Повседневные
  - Периодические
  - Разовые
- **Категория** — классификация по назначению:
  - Продукты
  - Рестораны
  - Такси
  - Образование
  - Путешествия
  - Другие категории
- **Описание** — подробная информация о транзакции:
  - Товары/услуги
  - Источник дохода
  - Контрагент
  - Дополнительные детали

### 2. Автоматическое извлечение данных

Бот анализирует сообщения пользователя и автоматически:
- Извлекает информацию о транзакциях
- Структурирует данные согласно формату таблицы
- Заполняет все необходимые поля
- Классифицирует транзакции по типу и категории

### 3. Отчетность

По запросу пользователя бот предоставляет:
- Текущий баланс
- Отчеты о доходах и расходах
- Аналитику по категориям и типам

### 4. Обработка различных типов сообщений

#### Текстовые сообщения
- Обработка естественного языка
- Извлечение информации о тратах и заработке
- Понимание контекста разговора

#### Голосовые сообщения
- Транскрибация голоса в текст
- Извлечение транзакций из транскрипции
- Обработка как текстовых сообщений

#### Изображения
- Обработка фотографий чеков
- Распознавание текста (OCR)
- Извлечение данных о покупках

---

## Технологический стек

- **Платформа:** Telegram Bot
- **LLM:** Для понимания естественного языка и извлечения данных
- **Хранение:** In-memory (на этапе MVP)
- **Транскрибация:** Speech-to-Text API
- **OCR:** Для обработки изображений чеков

## Сценарии использования

### Пример 1: Текстовое сообщение
```
Пользователь: "Купил продукты в Пятерочке на 2500 рублей"
Бот: "✅ Записал расход: 2500₽ на продукты"
```

### Пример 2: Голосовое сообщение
```
Пользователь: [голосовое] "Сегодня поужинал в ресторане, потратил три тысячи"
Бот: "✅ Записал расход: 3000₽ в категории рестораны"
```

### Пример 3: Фото чека
```
Пользователь: [фото чека из кафе]
Бот: "✅ Обработал чек: 1250₽ в категории рестораны"
```

### Пример 4: Запрос баланса
```
Пользователь: "Покажи мой баланс за месяц"
Бот: [предоставляет детальный отчет]
```

## Этапы разработки

1. **MVP:** Базовый бот с обработкой текстовых сообщений
2. **Расширение:** Добавление голосовых сообщений
3. **Визуальное распознавание:** Обработка изображений чеков
4. **Аналитика:** Расширенные отчеты и инсайты
5. **Оптимизация:** Улучшение точности и UX

---

## Ограничения MVP

- Хранение данных только в памяти (без персистентности)
- Базовая категоризация
- Простые отчеты
- Без экспорта данных
- Без интеграции с банками
